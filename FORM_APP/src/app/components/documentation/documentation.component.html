EventService

El servicio EventService se encarga de realizar peticiones HTTP a la API de eventos. Este servicio es inyectable y se proporciona en la raíz de la aplicación.

Métodos:
  • getInfo()
    • Descripción: Obtiene información general de la API de eventos
    • Parámetros: Ninguno.
    • Retorno: Un observable que emite la respuesta de la API en formato any.
    • id: Identificador único del evento (generado aleatoriamente).
  • postEvent()
    • El método postEvent utiliza el objeto HttpClient de Angular para realizar una petición HTTP POST a la URL configurada en la variable environment.MOCKS. El cuerpo de la petición es el objeto request que se pasa como parámetro.
    • La interfaz IEvent define la estructura del objeto que se pasa como parámetro al método postEvent(). La interfaz debe contener las propiedades necesarias para crear un nuevo evento en la API de eventos.
AlertService

El servicio AlertService proporciona una forma sencilla de mostrar alertas y mensajes de confirmación en la aplicación. Utiliza la biblioteca SweetAlert2 para mostrar las alertas.

Métodos:
  • warning(title: string, text: string)
  • error(title: string, text: string)
  • success(title: string, text: string)
  • question(title: string, text: string, confirmButtonText: string): Devuelve una promesa de tipo SweetAlertResult que se resuelve con el resultado de la alerta.
  • info(title: string, text: string)
  • EventService: Servicio que proporciona la información de los eventos.
  • AlertService: Servicio que proporciona métodos para mostrar alertas y notificaciones al usuario.
IEvent

La interfaz IEvent define la estructura de un objeto que representa un evento. Esta interfaz se utiliza para tipar los objetos que se pasan como parámetro al método postEvent del servicio EventService.

Propiedades:
  • "id": Tipo number, identificador único del evento.
  • "date": Tipo Date, fecha del evento.
  • "description": Tipo string, descripción del evento.
  • "type": Tipo string, tipo de evento.
IMenu

La interfaz IMenu define la estructura de un objeto que representa un elemento de menú. Esta interfaz se utiliza para tipar los objetos que se pasan como parámetro al componente SidebarComponent.

Propiedades:
  • "title": Tipo string, título del elemento de menú.
  • "icon": Tipo string, clase (NgBootstrap) de ícono del elemento de menú.
  • "title": Tipo string, ruta asociada al elemento de menú.
IFilter

La interfaz IFilter define la estructura de un objeto que representa un filtro de eventos. Esta interfaz se utiliza para tipar los objetos que se pasan como parámetro al componente ConsultEventComponent.

Propiedades:
  • "type": Tipo string, tipo de evento seleccionado.
  • "title": Tipo string, fecha de inicio seleccionada.
  • "title": Tipo string, fecha de fin seleccionada.
SidebarComponent

Este componente es responsable de renderizar la barra lateral de navegación en la aplicación. Muestra una lista de elementos de menú que permiten al usuario navegar entre diferentes secciones de la aplicación.

Propiedades:
  • menu: Arreglo de objetos que contiene la información de los elementos de menú.
    • title: Título del elemento de menú.
    • icon: Clase de icono del elemento de menú.
    • route: Ruta asociada al elemento de menú.
Uso:
  1. Para utilizar este componente, simplemente se debe importar en el módulo correspondiente y agregarlo a la plantilla HTML.
  2. Luego, se puede agregar el componente a la plantilla HTML instanciando el input menu. Donde menu es un arreglo de objetos que contiene la información de los elementos de menú.
CreateEventComponent

Este componente es responsable de crear nuevos eventos en la aplicación. Permite al usuario ingresar la descripción del evento y seleccionar el tipo de respuesta.

Propiedades:
  • eventForm: Objeto que almacena la información del evento a crear.
    • type: Tipo de evento (manual por defecto).
    • description: Descripción del evento.
    • date: Fecha del evento (fecha actual por defecto).
    • id: Identificador único del evento (generado aleatoriamente).
Métodos:
  • eventFormValid: Propiedad que indica si el formulario de evento es válido (la descripción no está vacía y se ha seleccionado un tipo de respuesta).
  • postEvent: Método que crea un nuevo evento en la base de datos. Muestra una alerta de confirmación antes de enviar la solicitud.
ConsultEventComponent

Este componente es responsable de consultar y filtrar eventos en la aplicación. Permite al usuario seleccionar un tipo de evento, fecha de inicio y fecha de fin para filtrar los resultados.

Propiedades:
  • filters: Objeto que almacena los valores de los filtros seleccionados por el usuario.
    • type: Tipo de evento seleccionado.
    • initDate: Fecha de inicio seleccionada.
    • endDate: Fecha de fin seleccionada.
  • eventsInfo: Arreglo de objetos que contiene la información de los eventos.
  • eventType: Arreglo de strings que contiene los tipos de eventos disponibles.
  • responseType: Arreglo de strings que contiene los tipos de respuestas disponibles.
  • headers: Arreglo de strings que contiene los encabezados de la tabla de eventos.
Métodos:
  • ngOnInit: Método que se ejecuta al inicializar el componente. Obtiene la información de los eventos mediante el método getEvents.
  • getEvents: Método que obtiene la información de los eventos mediante el servicio EventService.
  • resetFilter: Método que reinicia los valores de los filtros.
  • buildFilter: Método que construye un objeto con los valores de los filtros seleccionados.
  • filterInformation: Método que filtra la información de los eventos en función de los valores de los filtros seleccionados.
  • search: Método que realiza una búsqueda de eventos en función de los valores de los filtros seleccionados.
  • validFilters: Propiedad que indica si hay algún filtro seleccionado.
